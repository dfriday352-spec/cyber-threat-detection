# Complete Deployment Guide - Cyber Threat Detection System\n\n## Table of Contents\n1. Local Development Setup\n2. Model Training\n3. Docker Deployment\n4. Kubernetes Deployment\n5. Cloud Deployment (AWS)\n6. Monitoring & Troubleshooting\n\n---\n\n## Phase 1: Local Development Setup\n\n### 1.1 Clone and Setup\n```bash\ngit clone https://github.com/dfriday352-spec/cyber-threat-detection.git\ncd cyber-threat-detection\n```\n\n### 1.2 Create Virtual Environment\n```bash\npython -m venv venv\nsource venv/bin/activate  # Windows: venv\\Scripts\\activate\npip install -r requirements.txt\n```\n\n### 1.3 Start Docker Services\n```bash\n# Start PostgreSQL, Redis, and API\ndocker-compose -f docker-compose.dev.yml up -d\n\n# Verify services\ndocker-compose ps\n```\n\n---\n\n## Phase 2: Model Training\n\n### 2.1 Generate Training Data\n```bash\npython scripts/train_models.py\n```\n\nThis will:\n- Generate 5,000 synthetic security events\n- Extract 128 features per event\n- Split into train/test sets\n- Create baseline statistics\n\n### 2.2 Verify Models\n```bash\nls -la models/\n# Output shows trained models and configuration\n```\n\n### 2.3 Test API with Models\n```bash\n# Health check\ncurl http://localhost:8000/health\n\n# Analyze a threat\ncurl -X POST http://localhost:8000/api/v1/threats/analyze \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"source_ip\": \"192.168.1.100\",\n    \"destination_ip\": \"10.0.0.1\",\n    \"packet_count\": 5000,\n    \"byte_count\": 1000000,\n    \"protocol\": \"TCP\"\n  }'\n```\n\n---\n\n## Phase 3: Docker Deployment\n\n### 3.1 Build Docker Images\n```bash\n# Build API container\ndocker build -f docker/Dockerfile.api -t cyber-threat-api:latest .\n\n# Build detector container\ndocker build -f docker/Dockerfile.detector -t cyber-threat-detector:latest .\n\n# Build response container\ndocker build -f docker/Dockerfile.response -t cyber-threat-response:latest .\n```\n\n### 3.2 Run Full Stack\n```bash\n# Start all services (production setup)\ndocker-compose up -d\n\n# Check status\ndocker-compose ps\n\n# View logs\ndocker-compose logs -f\n```\n\n### 3.3 Scale Services\n```bash\n# Scale API to 3 instances\ndocker-compose up -d --scale api=3\n\n# Check running containers\ndocker ps | grep cyber-threat\n```\n\n---\n\n## Phase 4: Kubernetes Deployment\n\n### 4.1 Local Cluster Setup (Minikube)\n```bash\n# Start Minikube\nminikube start --cpus=4 --memory=4096\n\n# Verify cluster\nkubectl cluster-info\n\n# Enable metrics\nminikube addons enable metrics-server\n```\n\n### 4.2 Create Namespace and Secrets\n```bash\n# Create namespace\nkubectl create namespace threat-detection\n\n# Create database secret\nkubectl create secret generic threat-detection-secret \\\n  --from-literal=database-url='postgresql://threat_user:threat_password@postgres-service:5432/threat_detection' \\\n  -n threat-detection\n```\n\n### 4.3 Deploy Services\n```bash\n# Deploy Redis\nkubectl apply -f kubernetes/redis.yaml -n threat-detection\n\n# Deploy PostgreSQL\nkubectl apply -f kubernetes/postgres.yaml -n threat-detection\n\n# Wait for services to be ready\nkubectl wait --for=condition=available --timeout=300s \\\n  deployment/redis -n threat-detection\nkubectl wait --for=condition=available --timeout=300s \\\n  deployment/postgres -n threat-detection\n\n# Deploy API and services\nkubectl apply -f kubernetes/deployment.yaml -n threat-detection\n\n# Verify deployment\nkubectl get pods -n threat-detection\nkubectl get svc -n threat-detection\n```\n\n### 4.4 Access Application\n```bash\n# Port forward to local machine\nkubectl port-forward svc/cyber-threat-detection-service 8000:80 \\\n  -n threat-detection\n\n# Test API\ncurl http://localhost:8000/health\n```\n\n### 4.5 Monitor Deployment\n```bash\n# View logs\nkubectl logs -f deployment/cyber-threat-detection-api -n threat-detection\n\n# Check metrics\nkubectl top pods -n threat-detection\n\n# Describe pod for debugging\nkubectl describe pod <pod-name> -n threat-detection\n```\n\n---\n\n## Phase 5: Cloud Deployment (AWS)\n\n### 5.1 Push to Docker Registry\n```bash\n# Login to Docker Hub\ndocker login\n\n# Tag images with your registry\ndocker tag cyber-threat-api:latest yourname/cyber-threat-api:1.0.0\ndocker tag cyber-threat-detector:latest yourname/cyber-threat-detector:1.0.0\n\n# Push to registry\ndocker push yourname/cyber-threat-api:1.0.0\ndocker push yourname/cyber-threat-detector:1.0.0\n```\n\n### 5.2 Create AWS EKS Cluster\n```bash\n# Create cluster (takes 10-15 minutes)\naws eks create-cluster \\\n  --name cyber-threat-detection \\\n  --version 1.27 \\\n  --roleArn arn:aws:iam::YOUR_ACCOUNT_ID:role/eks-service-role \\\n  --resourcesVpcConfig subnetIds=subnet-xxx,subnet-yyy\n\n# Get cluster credentials\naws eks update-kubeconfig \\\n  --name cyber-threat-detection \\\n  --region us-east-1\n\n# Verify connection\nkubectl cluster-info\n```\n\n### 5.3 Deploy to EKS\n```bash\n# Create namespace\nkubectl create namespace threat-detection\n\n# Update kubernetes/deployment.yaml with your image URIs\n# Then deploy\nkubectl apply -f kubernetes/redis.yaml -n threat-detection\nkubectl apply -f kubernetes/postgres.yaml -n threat-detection\nkubectl apply -f kubernetes/deployment.yaml -n threat-detection\n\n# Get external IP\nkubectl get svc cyber-threat-detection-service -n threat-detection\n```\n\n---\n\n## Phase 6: Monitoring & Observability\n\n### 6.1 Prometheus Setup\n```bash\n# Add Prometheus Helm repository\nhelm repo add prometheus-community https://prometheus-community.github.io/helm-charts\nhelm repo update\n\n# Install Prometheus\nhelm install prometheus prometheus-community/kube-prometheus-stack \\\n  --namespace monitoring \\\n  --create-namespace\n```\n\n### 6.2 Grafana Setup\n```bash\n# Port forward to Grafana\nkubectl port-forward svc/prometheus-grafana 3000:80 -n monitoring\n\n# Access: http://localhost:3000\n# Default credentials: admin/prom-operator\n```\n\n### 6.3 View Metrics\n```bash\n# Check pod resource usage\nkubectl top pods -n threat-detection\n\n# Check node resource usage\nkubectl top nodes\n```\n\n---\n\n## Troubleshooting\n\n### Issue: Models not loading\n```bash\n# Retrain models\npython scripts/train_models.py\n\n# Check if files exist\nls -la models/\n```\n\n### Issue: API crashes\n```bash\n# Check logs\ndocker logs cyber_threat_api\n# or\nkubectl logs deployment/cyber-threat-detection-api -n threat-detection\n\n# Restart service\ndocker restart cyber_threat_api\n# or\nkubectl restart deployment/cyber-threat-detection-api -n threat-detection\n```\n\n### Issue: Database connection error\n```bash\n# Reset Docker services\ndocker-compose down -v\ndocker-compose -f docker-compose.dev.yml up -d\n\n# Or restart Kubernetes services\nkubectl delete pod postgres-0 -n threat-detection\nkubectl delete pod redis-0 -n threat-detection\n```\n\n### Issue: High memory usage\n```bash\n# Check resource limits\nkubectl describe node\n\n# Reduce replicas if needed\nkubectl scale deployment cyber-threat-detection-api --replicas=2 -n threat-detection\n```\n\n---\n\n## Production Checklist\n\n- [ ] Models trained and
